# Contribuyendo a Team-se-cayo-el-Server

¡Gracias por tu interés en contribuir a este proyecto universitario! 🎉

---

## 📌 Normas Generales
1. **Código limpio**: Sigue las mejores prácticas de programación en C#.
2. **Mensajes de commit claros**: Usa mensajes descriptivos (ej: `feat: add Producto model`).
3. **Pull Requests (PR)**: Cada PR debe estar asociado a un issue y debe ser revisado por al menos un miembro del equipo.
4. **Pruebas**: Si añades funcionalidad, incluye pruebas unitarias.

---

## 🛠 Requisitos para Contribuir
- Tener instalado **.NET 8 SDK**.
- Usar **Git** para control de versiones.
- Seguir la estructura del proyecto.

---

## 🚀 Cómo Contribuir
1. **Forkea el repositorio** y clónalo localmente.
2. **Crea una rama** para tu funcionalidad o corrección:
   ```bash
   git checkout -b feature/nueva-funcionalidad
   ```
3. **Realiza tus cambios** y haz commit:
   ```bash
   git commit -m "feat: descripcion de la funcionalidad"
   ```
4. **Empuja tus cambios** a tu fork:
   ```bash
   git push origin feature/nueva-funcionalidad
   ```
5. **Abre un Pull Request** en el repositorio principal.

---

## 📂 Estructura de Ramas
- `main`: Rama principal (solo releases estables).
- `feature/`: Para nuevas funcionalidades.
- `bugfix/`: Para correcciones de errores.
- `docs/`: Para mejoras en la documentación.

---

## 📝 Estándares de Código
- Usa **nombres descriptivos** para clases, métodos y variables.
- **Comenta tu código** cuando sea necesario.
- Sigue el estilo de código existente en el proyecto.

---

## 🤝 Revisión de Pull Requests
- Cada PR debe ser revisado por al menos **un miembro del equipo**.
- Se aceptarán PRs que:
  - No rompan el código existente.
  - Incluyan pruebas (si aplica).
  - Estén bien documentados.

---

## 📞 Contacto
Si tienes dudas, abre un **issue** en el repositorio o contacta al equipo.
